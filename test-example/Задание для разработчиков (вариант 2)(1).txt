Наша компания один из лидеров EDI рынка.
Основной смысл EDI это передача XML файлов с различной бизнес информацией из одной системы в другую с различной промежуточной обработкой.
На данный момент мы ищем кандидатов на продукт LinkServer, основное назначение которого интеграция между внутренними системами клиента 
и внешними платформами EDI провайдеров с различной прикладной обработкой данных в промежуточных точках (сравнение, анализ, трансформация, хранение). 

==== Задание для Разработчика ==== 
- Используемые технологии: Java 1.8, Spring Framework, XSLT, XML, приветствуется использование Apache CAMEL
Необходимо разработать приложение для приема входящего XML документа с последующей его обработкой и выгрузкой.
У клиента имеется xml файл (оригинал), который он выгружает из своей системы в определенный каталог (имитация работы по FTP).
Необходимо его загрузить в приложение, обработать и выгрузить.
Обработка: 
1) Загрузка оригинального файла из каталога Z (если файл прочитан, то удалять его из каталога)
2) Применение xslt преобразования к оригинального файлу
3) Сохранение оригинального и полученного в результате трансформации xml в БД (результирующий xml должен иметь ссылку на оригинальный файл)
4) Выгрузка полученного в результате трансформации xml в "каталог 1" (имя файла вычисляется на основании xpath по тегу OrderNumber)
5) Формирование zip архива состоящего из оригинального и полученного файлов и выгрузка архива в "каталог 2" (имя архива представляет собой имя оригинального файла + расширение zip)
6) Сохранение xmlей в базе данных по полям (id, OrderNumber, xml, date)
7) Разработка REST сервиса, для получения списка полученных документов прошедших через приложение, фильтр по OrderNumber // Список в каком виде??? Что понимается под фильтром???
8) Разработка REST сервиса, для скачивания документа xml по конкретному id (OrderNumber??) //Оригинального или результируещего????
9) Код должен быть покрыть unit тестами
